טיפוסים של משתנים

ל-RUST יש ​​טיפוסים רבים של משתנים המאפשרים לך לעבוד עם מספרים, תווים וכן הלאה. חלקם פשוטים, אחרים מסובכים יותר, ואפילו תוכלו ליצור טיפוסים משלכם.

טיפוסים פרימיטיביים

ל-RUST יש ​​סוגים משתנים פשוטים הנקראים:פרימיטיביים (פרימיטיביים = בסיסיים מאוד). נתחיל במספרים שלמים ובתווים. מספרים שלמים הם מספרים שלמים ללא נקודה עשרונית. ישנם שני סוגים של מספרים שלמים:

מספרים שלמים עם סימן,
מספרים שלמים ללא סימן.

סימנים פירושם " + " (סימן פלוס) ו- " - " (סימן מינוס), ולכן מספרים שלמים עם סימן יכולים להיות חיוביים או שליליים (למשל +8, -8). אך מספרים שלמים ללא סימן יכולים להיות חיוביים בלבד מכיוון שאין להם סימן.

המספרים השלמים עם סימן הם: i8, i16, i32, i64, i128 ו- isize. המספרים השלמים שללא סימן הם: u8, u16, u32, u64, u128, andize.

המספר אחרי ה- i או- u פירושו מספר הביטים עבור המספר, כך שמספרים עם יותר ביטים יכולים להיות גדולים יותר. 8 ביטים = בית אחד, אז i8 הוא בית אחד, i64 הוא 8 בתים, וכן הלאה. טיפוסי שלמים בגדלים גדולים יותר יכולים להכיל מספרים גדולים יותר. לדוגמה, u8 יכול להכיל עד 255, אבל u16 יכול להחזיק עד 65535. ו- u128 יכול להחזיק עד 340282366920938463463374607431768211455.

אז מה זה isize ושימושו? פירוש הדבר מספר הסיביות בסוג המחשב שלך. (מספר הסיביות במחשב שלך נקרא ארכיטקטורה של המחשב שלך.) אז גודל ושימוש בגודל של מחשב 32 סיביות זה כמו i32 ו- u32, וגודל ושימוש בגודל של מחשב 64 סיביות הוא כמו i64 ו- u64.

ישנן סיבות רבות לסוגי המספרים השלמים השונים. סיבה אחת היא ביצועי מחשב: מספר קטן יותר של בתים מהיר יותר לעיבוד. לדוגמה, המספר -10 בתור i8 הוא 11110110, אך בתור i128 הוא 1111111111111111111111111111111111111111111111111111111111111111111111111111111111111111111111111111111111111111110110. אבל הנה כמה שימושים אחרים:

תווים ב-RUST נקראים char. לכל תו יש מספר: האות A היא מספר 65, ואילו התו 友 ("חבר" בסינית) הוא מספר 21451. רשימת המספרים נקראת "Unicode ."Unicode משתמש במספרים קטנים יותר עבור תווים המשמשים יותר, כמו A עד Z, ​​או ספרות 0 עד 9, או רווח.

fn main() {
    let first_letter = 'A';
    let space = ' '; // A space inside ' ' is also a char
    let other_language_char = 'Ꮔ'; // Thanks to Unicode, other languages like Cherokee display just fine too
    let cat_face = '😺'; // Emojis are chars too
}

תווים ב-RUST נקראים char. לכל תו יש מספר: האות A היא מספר 65, ואילו התו 友 ("חבר" בסינית) הוא מספר 21451. רשימת המספרים נקראת "Unicode ."Unicode משתמש במספרים קטנים יותר עבור תווים המשמשים יותר, כמו A עד Z, ​​או ספרות 0 עד 9, או רווח.

התווים הנפוצים ביותר ממוספרים עד- 256 , והם יכולים להשתלב ב- u8 . זכור, u8 הוא 0 בתוספת כל המספרים עד 255 , עבור 256 בסך הכל. משמעות הדבר היא ש-RUST יכולה להמיר בבטחה u8 לתו, באמצעות- as. ("ייצוג u8 בתור char" פירושו "העמדת פנים u8 הוא char")

הצגה עם as שימושית כי RUST מאוד קפדנית. המהדר תמיד צריך לדעת את סוג המשתנה, ולא יאפשר לך להשתמש בשני סוגים שונים יחד גם אם שניהם מספרים שלמים. לדוגמה, זה לא יעבוד:

fn main() { // main() is where Rust programs start to run. Code goes inside {} (curly brackets)

    let my_number = 100; // We didn't write a type of integer,
                         // so Rust chooses i32. Rust always
                         // chooses i32 for integers if you don't
                         // tell it to use a different type
			 
    println!("{}", my_number as char); // ⚠️
}

וזאת הסיבה:

error[E0604]: only `u8` can be cast as `char`, not `i32`
 --> src\main.rs:3:20
  |
3 |     println!("{}", my_number as char);
  |                    ^^^^^^^^^^^^^^^^^

למרבה המזל אנחנו יכולים לתקן את זה בקלות עם as. אנחנו לא יכולים להציג את i32 כתו, אבל אנחנו יכולים להציג i32 כ- u8. ואז נוכל לעשות את אותו הדבר מ- u8 לתו. אז בשורה אחת אנו משתמשים בכדי להפוך את המספר שלי ל- u8 , ושוב להפוך אותו לתו. עכשיו זה יעבוד:

fn main() {
    let my_number = 100;
    println!("{}", my_number as u8 as char);
}

זה מדפיס את d כי ממוספר במקום ה-100.

הדרך הקלה יותר, עם זאת, היא רק לומר ל-RUST ש- my_number הוא u8 . כך אתה עושה את זה:

fn main() {
    let my_number: u8 = 100; //  change my_number to my_number: u8
    println!("{}", my_number as char);
}

אז אלה שתי סיבות לסוגי המספרים השונים ב-RUST. הנה סיבה נוספת: השימוש usize הוא הגודל שמשמשת Rust לאינדקסציה. (אינדקס פירושו "איזה פריט הוא הראשון", "איזה פריט הוא השני" וכו ') usize הוא הגודל הטוב ביותר לאינדקס כי:

אינדקס לא יכול להיות שלילי, ולכן הוא צריך להיות מספר עם- u
זה צריך להיות גדול, כי לפעמים אתה צריך להוסיף הרבה דברים לאינדקס, אבל
זה לא יכול להיות u64 מכיוון שמחשבי 32 סיביות אינם יכולים להשתמש ב- u64.

אז Rust משתמש ב usize כך שהמחשב שלך יוכל לקבל את המספר הגדול ביותר לאינדקס שהוא יכול לקרוא.

בואו ללמוד עוד קצת על char . ראית תו הוא תמיד תו יחיד ומשתמש '' במקום "".

כל התווים הם 4 בתים. הם 4 בתים מכיוון שחלק מהתווים במחרוזת הם יותר מבית אחד. אותיות בסיסיות שתמיד היו במחשבים הן 1 בתים, תווים מאוחרים יותר הם 2 בתים, ואחרים הם 3 ו -4. תו צריך להיות 4 בתים כדי שהוא יכול להכיל כל סוג של תו.

אנו יכולים להשתמש ב- .len () כדי לראות זאת בעצמנו:

fn main() {
    println!("{}", "a".len()); // .len() gives the size in bytes
    println!("{}", "ß".len());
    println!("{}", "国".len());
    println!("{}", "𓅱".len());
}

זה ידפיס:

1
2
3
4

ניתן לראות ש- a הוא בת אחד, הגרמני ß הוא שניים, היפני 国 הוא שלושה, והמצרי הקדום 𓅱 הוא 4 בתים.

fn main() {
    let slice = "Hello!";
    println!("Slice is {} bytes.", slice.len());
    let slice2 = "안녕!"; // Korean for "hi"
    println!("Slice2 is {} bytes.", slice2.len());
}

זה ידפיס:

Slice is 6 bytes.
Slice2 is 7 bytes.

האורך של slice הוא 6 תווים ו -6 בתים, אך אורך slice2 הוא 3 תווים ו -7 בתים.

אם .len () נותן את הגודל בבתים, מה לגבי הגודל בתווים? נלמד על שיטות אלה בהמשך, אך אתה רק יכול לזכור ש .chars (). Count () יעשה את זה. .chars (). count () הופך את מה שכתבת לדמויות ואז סופר כמה יש.

fn main() {
    let slice = "Hello!";
    println!("Slice is {} bytes and also {} characters.", slice.len(), slice.chars().count());
    let slice2 = "안녕!";
    println!("Slice2 is {} bytes but only {} characters.", slice2.len(), slice2.chars().count());
}

זה ידפיס:

Slice is 6 bytes and also 6 characters.
Slice2 is 7 bytes but only 3 characters.
